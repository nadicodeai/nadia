# Assertions for 0002-rebrand-install-urls.patch
#
# After `make build`, the install URLs and clone/issues URLs in
# README.md, README.zh-CN.md, and CONTRIBUTING.md must point at
# nadicodeai/argo (the rebrand target). The two final LICENSE links
# are intentionally NOT rewritten — they must still point at the
# upstream NousResearch/hermes-agent LICENSE file so attribution
# survives the rename engine pass.
#
# Additionally (M3.1, IU-AC-9 static gate), the renamed
# `argo_cli/main.py` `OFFICIAL_REPO_URLS` set (defined at upstream
# `hermes_cli/main.py:7319-7332`, compared against by `_is_fork()`)
# MUST list `nadicodeai/argo` in both HTTPS and SSH forms — otherwise
# every `argo update` from a release-branch install would print
# "⚠ Updating from fork: ..." to customers. Pinned here (not in a
# dedicated patch) because the rename engine already rewrites the
# set cleanly; the assertion catches any future regression where
# the rewrite stops covering this slot.
